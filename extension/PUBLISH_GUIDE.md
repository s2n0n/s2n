# 크롬 웹 스토어 업로드 및 공식 ID 교체 가이드 (비공개 배포)

이 프로젝트는 터미널 한 줄로 손쉽게 프론트엔드를 패키징할 수 있도록 스크립트가 구성되어 있습니다. 추후 웹 스토어 비공개 출시를 통해 누구나 다운로드 받을 수 있게 하려면 아래 단계를 따라주세요.

## 1. 패키지 압축 (.zip 생성)
먼저 아래 명령어를 실행하면 `dist` 폴더 빌드 후 자동으로 크롬 웹 스토어 업로드용 `s2n-extension.zip` 파일이 `extension` 폴더 안에 생성됩니다.
```bash
cd extension
npm run build:zip
```

## 2. 크롬 웹 스토어 개발자 대시보드 업로드
- [크롬 웹 스토어 개발자 대시보드](https://chrome.google.com/webstore/devconsole) 에 접속합니다.
  *(※ 참고: 구글 정책상 개발자 계정 최초 등록 시 단 1회, 5달러의 가입비용이 발생합니다. 그 이후의 확장 프로그램 배포/업데이트 등은 개수 상관없이 영구 전면 무료입니다.)*
- 우측 상단 **"새 항목(New Item)"** 버튼을 클릭하고 방금 만든 `s2n-extension.zip`을 업로드합니다.
- 좌측 탭에서 스토어 정보(이름, 설명, 캡처 화면 1개 포함 권장)를 대략적으로 입력합니다.
- **"공개 범위(Visibility)"** 를 🌐 전체 공개(Public) 대신 **🔒 비공개(Private / 링크가 있는 사람만)** 로 설정합니다.
- 심사 제출(Publish) 버튼을 누릅니다. (비공개는 심사가 보통 당일 내로 빠르게 처리됩니다.)

## 3. 공식 Extension ID 교체 작업 (매우 중요 ⭐️)
웹 스토어에 업로드를 마치면, 크롬 웹 스토어가 이 프로그램의 영구적인 **공식 Extension ID**를 발급해줍니다. 

사용자들이 이 스토어에서 받은 버전과 `s2n install-host` 명령어가 서로 문제없이 통신하도록 만들려면, 스토어에서 발급받은 **공식 ID**로 CLI 소스코드를 한 번만 업데이트(치환)해주셔야 합니다.

1. 개발자 대시보드 URL 혹은 목록 썸네일에 적혀있는, **알파벳 소문자 32글자로 된 사용자의 공식 Extension ID**를 복사합니다.
2. 로컬 프로젝트의 `s2n/s2nscanner/cli/runner.py` 파일 내 `install-host` 명령어 위쪽에 적혀있는 `default="..."` 문자열을 방금 복사한 새로운 공식 ID 알파벳으로 덮어씌워 줍니다.
