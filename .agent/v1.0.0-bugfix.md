**[BUG-01] `logger.py` — `get_logger()` 로거 이름 이중 점 버그 + `setLevel()` 중복 호출**

- **파일**: `s2n/s2nscanner/logger.py`
- **문제 1**: `f"{LOGGER_NAME}.{'.' + name if name else ''}"` 이 표현식에서 name이 있을 때 `"s2n." + "." + "http_client"` 가 되어 `"s2n..http_client"`(점 두 개)가 됩니다. name이 None이면 `"s2n."`(후행 점)이 됩니다. 로거 계층 구조가 완전히 깨집니다.
- **문제 2**: `logger.setLevel(...)` 를 같은 인자로 두 줄 연속 호출합니다.
- **수정**: `return logging.getLogger(f"{LOGGER_NAME}.{name}" if name else LOGGER_NAME)` 으로 수정, 중복 `setLevel()` 한 줄 제거

**[BUG-02] `dvwa_adapter.py` — `authenticate()` 로그인 실패해도 항상 성공 반환**

- **수정**: 로그인 성공 시에만 `return (user, pw)`, 실패 시 `continue` 또는 `return None`. `print()` → `self.logger` 교체

- **문제**: 모듈 레벨 독립 함수인데 `self: ClassVar` 파라미터를 받고 있습니다. `ClassVar`는 클래스 변수를 표시하는 type annotation이지, 파라미터 타입이 아닙니다. 함수가 `self`를 받는다는 것도 일반 함수로선 어색합니다.
- **수정**: `self` 파라미터 제거하고 필요한 값만 명시적으로 받도록 시그니처 정비, 또는 플러그인 Mixin으로 이동
