# s2n v1.0.0 Plugin: Autobot (봇 탐지 플러그인)

## 개요

**목적**: 타겟 URL이 크롤러/스크립트 봇의 행위를 탐지·차단하는지 검증하는 플러그인.  
**판정 기준**: 봇 행위 시도 → 차단당하면 `PASS`, 차단 안 당하면 `FAIL` (취약점 Finding 생성).

---

## 설계 원칙

- **봇 행위는 undeterministic**: 앱마다 목적이 다르므로 `BotBehavior` 추상 인터페이스로 분리하고, 구체 행위는 `Strategy` 패턴으로 주입한다.
- **Chromium 기반**: Selenium WebDriver (headless Chromium)로 실제 브라우저 환경에서 행위를 시뮬레이션한다. 기존 `brute_force_selenium.py`의 `setup_driver()` 패턴을 재사용한다.
- **판정 근거 명시**: 단순 HTTP 상태 코드만이 아닌, 응답 본문·리디렉션·CAPTCHA·IP 차단 등 복합 신호로 차단 여부를 판단한다.

---

## 봇 행위 추상화 (`autobot_behaviors.py`)

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass
from selenium.webdriver.remote.webdriver import WebDriver

@dataclass
class BehaviorResult:
    behavior_name: str
    was_blocked: bool          # True → 차단 확인 (PASS 기여)
    evidence: str              # 차단/통과 근거 텍스트
    raw_status: Optional[int]  # HTTP 상태 코드 (가능하면)

class BotBehavior(ABC):
    name: str

    @abstractmethod
    def execute(self, driver: WebDriver, target_url: str) -> BehaviorResult:
        """봇 행위를 실행하고 차단 여부를 반환."""
        ...
```

### 기본 구현체

| 클래스 | `name` | 행위 설명 |
|---|---|---|
| `RapidCrawlBehavior` | `rapid_crawl` | 1초 내 20개 링크 연속 GET (delay 없음) |
| `HeadlessSignalBehavior` | `headless_signal` | `navigator.webdriver=true` 상태 그대로 요청 |
| `NoUserInteractionBehavior` | `no_user_interaction` | 마우스 이동·스크롤 없이 즉시 폼 Submit |
| `RepetitiveQueryBehavior` | `repetitive_query` | 동일 검색어를 짧은 간격으로 재요청 |

> **확장**: 새 행위는 `BotBehavior`를 상속하고 `behaviors` key에 이름을 등록하면 된다.

## 차단 판정 (`autobot_detector.py`)

차단(block) 신호 우선순위 (하나라도 해당하면 `was_blocked=True`):

1. HTTP `429 Too Many Requests` 또는 `403 Forbidden`
2. 응답 본문에 CAPTCHA 키워드 포함 (`recaptcha`, `hcaptcha`, `captcha`, `robot`, `automated`)
3. Selenium 세션이 다른 도메인으로 리디렉트 (예: Cloudflare challenge 페이지)
4. `document.title` 또는 `window.location`에서 차단 페이지 패턴 감지

## `autobot_main.py` 실행 흐름

```
run(plugin_context)
│
├─ 1. resolve_target_url()                 # helper.py 공통 함수 사용
├─ 2. _request_user_confirmation()        # brute_force 패턴 그대로 (accept_risk 지원)
├─ 3. setup_driver(headless)              # autobot_driver.py → brute_force_selenium.setup_driver 재사용
├─ 4. behaviors = _load_behaviors(names)  # custom_params["behaviors"] 기반 팩토리
├─ 5. for behavior in behaviors:
│      result = behavior.execute(driver, target_url)
│      if not result.was_blocked → candidate finding 축적
├─ 6. driver.quit()                        # finally 블록
├─ 7. 판정:
│      차단되지 않은 행위 수 >= block_threshold
│        → Finding 생성 (Severity.HIGH, "Bot Detection Absent")
│      아니면 → findings 없음 (PASS)
└─ 8. PluginResult 반환
```

### `PluginStatus` 매핑

| 상황 | status |
|---|---|
| accept_risk=False + 사용자 N 입력 | `SKIPPED` |
| driver 초기화 실패 | `FAILED` |
| 일부 행위 실행 중 예외, finding 있음 | `PARTIAL` |
| 정상 완료 | `SUCCESS` |

## 안전성 고려사항

- **법적 책임 고지**: `brute_force` 플러그인과 동일하게 `_request_user_confirmation()` 필수.
- **세션 정리**: `finally` 블록에서 반드시 `driver.quit()`. 서버 측 세션 잔류 방지를 위해 로그아웃 URL이 있으면 시도.
- **요청 수 집계**: `requests_sent`를 각 행위에서 실제 전송한 수로 누적해 `PluginResult`에 반영 ([SCAN-06] 동일 버그 방지).
- **페이로드 없음**: 이 플러그인은 exploit 페이로드를 전송하지 않으므로 `SAFETY-05` 유형 위험은 해당 없음.

# s2n v1.0.0 Plugin: Autobot (Bot Detection Plugin)

## Overview

**Purpose**: A plugin that verifies whether a target URL can detect and block crawler/script bot behavior.  
**Verdict**: Bot behavior attempted → blocked = `PASS`, not blocked = `FAIL` (generates a vulnerability Finding).

---

## Design Principles

- **Bot behavior is undeterministic**: Every application has different bot use-cases, so concrete behaviors are decoupled behind a `BotBehavior` abstract interface and injected via the **Strategy** pattern.
- **Chromium-based**: Uses Selenium WebDriver (headless Chromium) to simulate real browser behavior. Reuses the `setup_driver()` pattern from the existing `brute_force_selenium.py`.
- **Evidence-based verdict**: Block detection is determined not by HTTP status code alone, but by a composite of response body keywords, redirects, CAPTCHA challenges, and IP-block signals.

---

## Bot Behavior Abstraction (`autobot_behaviors.py`)

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass
from selenium.webdriver.remote.webdriver import WebDriver

@dataclass
class BehaviorResult:
    behavior_name: str
    was_blocked: bool          # True → block confirmed (contributes to PASS)
    evidence: str              # Text explaining block / pass reason
    raw_status: Optional[int]  # HTTP status code if available

class BotBehavior(ABC):
    name: str

    @abstractmethod
    def execute(self, driver: WebDriver, target_url: str) -> BehaviorResult:
        """Execute the bot behavior and return the block verdict."""
        ...
```

### Default Implementations

| Class | `name` | Behavior Description |
|---|---|---|
| `RapidCrawlBehavior` | `rapid_crawl` | GET 20 links in under 1 second (no delay) |
| `HeadlessSignalBehavior` | `headless_signal` | Request with `navigator.webdriver=true` exposed |
| `NoUserInteractionBehavior` | `no_user_interaction` | Submit a form immediately with no mouse movement or scroll |
| `RepetitiveQueryBehavior` | `repetitive_query` | Re-request the same search query at short intervals |

> **Extension**: Inherit `BotBehavior` and register the new behavior name under the `behaviors` key.

---

## Block Detection (`autobot_detector.py`)

Block signal priority — any one match sets `was_blocked=True`:

1. HTTP `429 Too Many Requests` or `403 Forbidden`
2. Response body contains CAPTCHA keywords (`recaptcha`, `hcaptcha`, `captcha`, `robot`, `automated`)
3. Selenium session redirected to a different domain (e.g., Cloudflare challenge page)
4. `document.title` or `window.location` matches a known block-page pattern

---

## `autobot_main.py` Execution Flow

```
run(plugin_context)
│
├─ 1. resolve_target_url()                 # shared helper.py function
├─ 2. _request_user_confirmation()        # same pattern as brute_force (accept_risk supported)
├─ 3. setup_driver(headless)              # autobot_driver.py → reuses brute_force_selenium.setup_driver
├─ 4. behaviors = _load_behaviors(names)  # factory based on custom_params["behaviors"]
├─ 5. for behavior in behaviors:
│      result = behavior.execute(driver, target_url)
│      if not result.was_blocked → accumulate candidate finding
├─ 6. driver.quit()                        # in finally block
├─ 7. verdict:
│      unblocked behavior count >= block_threshold
│        → create Finding (Severity.HIGH, "Bot Detection Absent")
│      otherwise → no findings (PASS)
└─ 8. return PluginResult
```

### `PluginStatus` Mapping

| Condition | Status |
|---|---|
| `accept_risk=False` + user inputs N | `SKIPPED` |
| Driver initialization fails | `FAILED` |
| Exception during some behaviors, findings exist | `PARTIAL` |
| Normal completion | `SUCCESS` |

## Safety Considerations

- **Legal disclaimer**: `_request_user_confirmation()` is mandatory, identical to the `brute_force` plugin pattern.
- **Session cleanup**: `driver.quit()` must be called in a `finally` block. If a logout URL is available, attempt logout to avoid zombie server-side sessions.
- **Request count tracking**: Accumulate `requests_sent` from each behavior's actual HTTP requests and reflect them in `PluginResult` (prevents the same bug as [SCAN-06]).
- **No exploit payloads**: This plugin does not send exploit payloads, so [SAFETY-05]-type risks do not apply.
